PAPERLENS
Build a production-grade Agentic Literature Review Engine from scratch.
PaperLens is a multi-agent system that helps researchers discover, analyze, and synthesize scientific papers. Given a research question, it automatically retrieves papers from Semantic Scholar's 200M+ corpus, parses full-text PDFs, performs hybrid search, synthesizes findings across multiple papers, detects contradictions between studies, self-critiques output quality via a reflection loop, and builds persistent research memory across sessions — all orchestrated by a LangGraph state machine, served via FastAPI, and monitored with Langfuse.

What PaperLens Does
You ask: "What is the current evidence on transformer architectures for time-series forecasting?"

PaperLens automatically:
1. Discovers papers daily from Semantic Scholar (200M+ papers, all disciplines)
2. Downloads & parses full-text PDFs using Docling (tables, sections, references)
3. Indexes with hybrid search (BM25 keywords + 1024-dim Jina embeddings + RRF fusion)
4. Retrieves the most relevant chunks, with an agent that rewrites poor queries automatically
5. Synthesizes findings across multiple papers — themes, agreements, and gaps
6. Detects contradictions — "Paper A says X, Paper B says Y, possibly because..."
7. Self-critiques the answer quality via a reflection loop (re-generates if score < 0.7)
8. Remembers your prior research — the next query builds on what you've already explored

DESIGN PATTERNS
Production patterns. Not tutorials.
Every architectural decision follows established software engineering patterns. Week 8 adds two advanced patterns: Evaluator-Optimizer and Memento.
Factory Pattern
Every service has factory.py
Decouples construction from use. Enables dependency injection and test mocking.
Repository Pattern
PaperRepository
Isolates database queries from business logic. Clean data access layer.
Dependency Injection
FastAPI Depends(), LangGraph Context
Testability, loose coupling, runtime configuration.
Strategy Pattern
OpenSearch (BM25 / hybrid / vector)
Swap search algorithms without changing callers.
Abstract Interface
BaseDatabase, BaseRepository
Database backend is swappable (PostgreSQL → any other).
Singleton via LRU
@lru_cache factories
Reuse expensive connections (DB, search, LLM).
Context Manager
DB sessions, Langfuse spans
Guaranteed resource cleanup, even on exceptions.
State Machine
LangGraph AgentState
Explicit workflow transitions with typed state.
Graceful Degradation
Embeddings→BM25, Cache→generate, Memory→skip
Partial failures never crash the entire system.
Evaluator-Optimizer W8
Critic node reflection loop
Scores output, provides feedback, re-generates until quality threshold met.
Memento Pattern W8
Research memory persistence
System remembers prior research state across sessions via Redis.
Observer Pattern
Langfuse tracing throughout
Non-intrusive monitoring of every pipeline stage.

WEEK 8 — NEW
PaperLens Extensions
This is where PaperLens goes beyond the course. Four new agent nodes transform single-paper Q&A into a true literature review engine. Plus an evaluation harness to prove it works.
New Agent Nodes (4)
1
🔬 Synthesis Node
Groups chunks by paper, identifies key themes, agreements, and research gaps across multiple papers. Outputs structured SynthesisResult with narrative + citations.
2
⚔️ Contradiction Node
Compares findings across papers. Detects conflicting conclusions, methodological disagreements. Explains WHY papers disagree (different populations, methods, timeframes). Only runs with ≥2 papers.
3
🎯 Critic Node (Reflection Loop)
Scores answer quality on 5 criteria (faithfulness, completeness, citations, coherence, contradiction handling). If score < 0.7, sends feedback and retriggers generation. Max 2 retries. This creates the measurable metric: "Improved faithfulness from X% to Y%."
4
🧠 Memory Node (Persistent Research)
Two phases: (1) Retrieve prior research for the session, inject as context. (2) Store current synthesis as new memory entry. Uses Redis with 90-day TTL. Graceful skip if Redis unavailable.
